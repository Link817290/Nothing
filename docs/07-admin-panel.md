# 07 — 管理面板

## 两个面板

| 面板 | 谁用 | 用途 | 版本 |
|------|------|------|------|
| 用户 Dashboard | 注册用户 | 收件箱、发件箱、Token 管理、设置 | v1 |
| 管理后台 | 平台运营 | 用户管理、全局统计、审核域名 | v2 |

## 技术选型

| 层 | 选择 | 理由 |
|---|------|------|
| 前端 | React + Vite | 纯 SPA，构建为静态文件，Caddy 直接托管，零运行时内存 |
| UI | Tailwind + shadcn/ui | 快速出界面，不需要设计师 |
| 路由 | react-router | SPA 客户端路由 |
| 认证 | GitHub OAuth（复用注册流程） | 用户已有账号，直接登录 |
| 部署 | 静态文件由 Caddy 托管，API 独立服务 | 前后端彻底分离 |

### Messages 页

消息列表，支持：
- 按未读/全部筛选
- 按项目筛选
- 按标签筛选
- 搜索正文/发件人/主题
- 点击进入消息详情（看线程、下载附件）

主要用于**查看历史和管理**，日常收发还是在 Agent 里操作。

## 管理后台

`nothing.email/admin`（仅管理员可访问）

### 页面结构

```
/admin
├── /overview        全局统计
├── /users           用户管理
├── /users/:id       用户详情
├── /domains         域名审核
├── /messages        消息监控（总量、趋势）
├── /reports         垃圾举报处理
└── /system          系统状态（邮件队列、磁盘、连接数）
```

### Overview 页


### Users 页

用户列表，支持：
- 搜索用户（handle / GitHub）
- 按套餐筛选
- 按信誉分筛选
- 操作：查看详情、冻结、删除

### Reports 页

处理垃圾举报：
- 查看被举报消息内容
- 确认/驳回举报
- 对用户执行处罚（警告 / 降信誉 / 冻结）

## v1 范围

| 页面 | v1 是否做 | 理由 |
|------|----------|------|
| Inbox | 做 | 用户需要在网页看收件箱 |
| Sent | 做 | 用户需要在网页看发件箱 |
| Tokens | 做 | 核心功能，创建/吊销 Token |
| Settings | 做 | 基础设置，账号管理 |
| Overview | 不做 | v1 不做用量统计图表 |
| Projects | 不做 | CLI 里看 |
| Domains | 不做 | v1 不做自定义域名 |
| Billing | 不做 | v1 全免费 |
| Admin 后台 | 不做 | v2 做，v1 直接查数据库 |
